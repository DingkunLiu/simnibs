# 动态文件名方案

## 1. 背景

当前代码中存在多处硬编码的文件名，导致：
- 头模生成结果固定为 `model.msh`，无法根据 subject ID 动态命名
- 正向仿真/优化结果均为 `TI_max_TI.nii.gz`（由 `interpolate_to_volume` 函数固定生成），同一 `output_dir` 中仿真和优化的结果会相互覆盖

## 2. 目标

- 从 `dir_path`（`m2m_xxx` 形式）中提取 `xxx` 作为 `subid`
- 头模生成结果改为 `{subid}.msh` 形式（如 `ernie.msh`）
- 正向仿真结果改为 `{subid}_simulation_max_TI.nii.gz` 形式
- 优化结果改为 `{subid}_optimization_max_TI.nii.gz` 形式

## 3. 修改方案

### 3.1 统一 subid 提取逻辑

**文件**: `simnibs/utils/file_finder.py`

`SubjectFiles` 类已实现 `subid` 的提取（通过正则 `m2m_(.+)`），**该文件无需修改**。

### 3.2 头模生成结果命名

**文件**: `neuracle/charm/mesh.py`

**现状**:
```python
output_msh_path = os.path.join(subject_dir, "model.msh")
```

**方案**: 改为使用 `{subid}.msh`
```python
output_msh_path = os.path.join(subject_dir, f"{subid}.msh")
```

需要修改的位置：
- 第 73 行：`output_msh_path`
- 第 81 行：调用 `write_head_mesh` 时的路径参数

### 3.3 正向仿真结果命名

**文件**: `neuracle/ti_simulation/run.py`

**现状**:
```python
mesh1_path = os.path.join(output_dir, f"{base_name}_TDCS_1_scalar.msh")
mesh2_path = os.path.join(output_dir, f"{base_name}_TDCS_2_scalar.msh")
```

这里的 `base_name` 来自 `session.fnamehead`（如 `ernie`），所以实际输出已经是 `ernie_TDCS_1_scalar.msh` 形式，**这部分已正确**。

### 3.4 TI 场计算结果命名

**文件**: `neuracle/ti_simulation/ti_calc.py`

**现状**:
```python
ti_mesh_path = os.path.join(output_dir, "TI.msh")
```

**方案**: 改为使用 `{base_name}_TI.msh`
```python
ti_mesh_path = os.path.join(output_dir, f"{base_name}_TI.msh")
```

### 3.5 导出 NIfTI 结果命名

**文件**: `neuracle/utils/ti_export.py`

**现状**:
```python
nifti_path = os.path.join(output_dir, "TI.nii.gz")
# interpolate_to_volume 会在文件名后加 _field_name，
# 所以实际输出为 TI_max_TI.nii.gz
```

**方案**: 支持传入前缀参数，改为 `{prefix}_TI_{field_name}.nii.gz`

函数签名修改为：
```python
def export_ti_to_nifti(
    msh_path: str,
    output_dir: str,
    reference: str,
    field_name: str,
    prefix: str = "",  # 新增参数
) -> str:
```

调用处传入 prefix：
- 正向仿真时: `prefix = f"{subid}_simulation"`
- 优化时: `prefix = f"{subid}_optimization"`

**影响范围**: 所有调用 `export_ti_to_nifti` 的地方都需要传递 `prefix` 参数：
- `neuracle/ti_simulation/demo/ti_simulation_demo.py`
- `neuracle/ti_optimization/demo/ti_max_optimize_demo.py`
- `neuracle/ti_optimization/demo/ti_mean_optimize_demo.py`
- `neuracle/ti_optimization/demo/ti_focality_optimize_demo.py`
- `neuracle/ti_optimization/demo/ti_focality_inv_optimize_demo.py`
- `neuracle/rabbitmq/scheduler.py`

### 3.6 优化结果命名（OSS 路径）

**文件**: `neuracle/rabbitmq/scheduler.py`

**现状**:
```python
# 正向仿真上传（line 496）
f"{normalize_dir_path(params.dir_path)}_TI_simulation_{task_id}/TI_max_TI.nii.gz"
# 优化上传（line 706）
f"{normalize_dir_path(params.dir_path)}_TI_optimization_{task_id}/TI_max_TI.nii.gz"
```

**方案**: 从 `params.dir_path` 提取 `subid`，改为：
```python
subid = re.search("m2m_(.+)", params.dir_path).group(1)
# 正向仿真上传
f"{normalize_dir_path(params.dir_path)}_TI_simulation_{task_id}/{subid}_simulation_max_TI.nii.gz"
# 优化上传
f"{normalize_dir_path(params.dir_path)}_TI_optimization_{task_id}/{subid}_optimization_max_TI.nii.gz"
```

### 3.7 RabbitMQ 消息路径

**文件**: `neuracle/rabbitmq/demo/simnibs_side_demo.py`

**现状**:
```python
result = {
    "TI_file": f"/data/{msg_id}/simulation_result.nii.gz",
}
```

**方案**: 改为
```python
result = {
    "TI_file": f"/data/{msg_id}/{subid}_simulation_max_TI.nii.gz",
}
```

## 5. 注意事项

1. **subid 传递**: 需要确保 `subid` 能从 `SubjectFiles` 传递到各个仿真模块，可能需要：
   - 通过函数参数传递
   - 通过 session 对象传递
   - 或在各个模块中重新从 `dir_path` 解析

2. **interpolate_to_volume 限制**: 该函数会在输出文件名后自动加 `_field_name`，所以传 `TI.nii.gz` 会生成 `TI_max_TI.nii.gz`。新的 prefix 参数需要在调用前拼接到输出路径中。

3. **测试**: 修改后按顺序运行 demo 验证：
   - 步骤1（头模生成）：`neuracle/charm/demo/charm_mesh_demo.py`，检查 `m2m_ernie/ernie.msh` 是否生成
   - 步骤2（正向仿真 + 逆向仿真，可并行）：`neuracle/ti_simulation/demo/ti_simulation_demo.py` 和 `neuracle/ti_optimization/demo/ti_focality_optimize_demo.py`，分别检查输出目录中是否有 `ernie_simulation_max_TI.nii.gz` 和 `ernie_optimization_max_TI.nii.gz`
